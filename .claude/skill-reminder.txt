IMPORTANT: keploy repo — mandatory skills

This repository (`keploy/keploy`) ships three repo-specific skills under
`.claude/skills/`. They are NOT optional review steps. Trigger them
AUTOMATICALLY as described below — do NOT wait for the user to ask.

- keploy-e2e-test
  INVOKE AUTOMATICALLY after implementing any change that touches:
    * CLI flags or commands (`cli/`, `main.go`)
    * record / replay pipelines (`pkg/service/record`, `pkg/service/replay`,
      `pkg/service/orchestrator`)
    * proxy / agent hooks (`pkg/core/proxy`, `pkg/agent/hooks`)
    * on-wire protocol handlers (`pkg/models/<protocol>`, `pkg/core/proxy/integrations`)
    * on-disk YAML format (`pkg/platform/yaml`)
    * matching / coverage / reports (`pkg/matcher`, `pkg/service/report`,
      `pkg/platform/coverage`)
    * `config/config.go`
  Unit tests passing and `go build` succeeding are NOT sufficient evidence
  that the change works. Running the sample-app record/replay is. If you
  decide to skip e2e, name one of the skip reasons from the skill's own
  skip list (pure refactor, enterprise-only, CLI-output-only, docs-only,
  missing infra, user hasn't told you what changed). Do not invent a new
  skip reason.

- keploy-pr-workflow
  INVOKE AUTOMATICALLY before creating/updating a pull request or issue,
  before any push to a shared branch, and before composing any commit
  that will leave the local machine. It carries the required PR format,
  commit-message conventions, sign-off rules, and customer-data hygiene
  checks.

- keploy-docs
  INVOKE AUTOMATICALLY whenever a code change introduces, removes, or
  alters user-visible behavior — new CLI flag, changed default, new
  command, new config field, new on-disk format. Docs in
  `github.com/keploy/docs` must catch up in lockstep, not later.

Gate your own "done" state: do NOT report a task complete until the
applicable skills above have either been invoked or explicitly skipped
with a reason drawn from the skill's own skip list.
